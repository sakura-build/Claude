# 开票申请 - 分项开票合同列表 含税类型功能需求文档

---

## 1. 背景

当前分项开票合同列表中仅有含税单价、含税小计，缺少不含税价格字段。根据税务规范，同一张发票内所有行项的计税方式必须统一。

含税类型的判定依据是合同。进入开票表单时，含税类型已从合同自动带出到主表，因此分项开票明细表中的设备列应从一开始就按该类型过滤，操作人只能看到同类型设备，无需担心混选。

---

## 2. 需求概述

在分项开票合同列表（`mk_model_20250415uji22_d_s0q4u`）中增加不含税单价、不含税小计字段；含税类型由合同自动带出到主表（只读并必填），并根据含税类型控制对应价格列的显隐；明细表设备列按主表含税类型过滤，只显示匹配的设备。

---

## 3. 功能范围

| 范围 | 说明 |
|------|------|
| 主表 | 新增 `含税类型` 字段，只读、必填，由合同自动带出 |
| 分项开票明细表 | 新增不含税单价、不含税小计；调整含税单价精度；设备列按含税类型过滤；按含税类型控制四列价格显隐 |
| 整单开票明细表 | 不涉及 |

---

## 4. 交互流程

```
进入表单
    |
    | 合同含税类型已自动带出到主表 fd_col_taxType（含税 或 不含税）
    v
分项开票明细表：
  - 设备列已按含税类型过滤，仅显示同类型设备
  - 含税时：显示含税单价、含税小计，隐藏不含税单价、不含税小计
  - 不含税时：显示不含税单价、不含税小计，隐藏含税单价、含税小计
```

---

## 5. 修改清单

### 5.1 主表

**1. 新增字段：含税类型**

| 项目 | 配置 |
|------|------|
| 控件 ID | `fd_col_taxType` |
| 控件类型 | 下拉单选 |
| 选项 | `含税` = 0，`不含税` = 1 |
| 可编辑 | 关闭（只读） |
| 必填 | 是 |
| 赋值方式 | 进入表单时由合同自动带出 |

---

### 5.2 分项开票明细表（`mk_model_20250415uji22_d_s0q4u`）

**2. 新增字段：不含税单价**

| 项目 | 配置 |
|------|------|
| 控件类型 | 数值 |
| 小数位数 | 4 位 |
| 显隐规则 | 可见当 `主表.fd_col_taxType == 1`（不含税） |

**3. 新增字段：不含税小计**

| 项目 | 配置 |
|------|------|
| 控件类型 | 数值 |
| 小数位数 | 2 位 |
| 计算公式 | `不含税单价 x 数量` |
| 显隐规则 | 可见当 `主表.fd_col_taxType == 1`（不含税） |

**4. 新增 / 调整字段：本次开票金额**

| 项目 | 配置 |
|------|------|
| 控件类型 | 数值 |
| 计算公式 | 含税时 `含税单价 x 数量`；不含税时 `不含税单价 x 数量`（IF 条件公式区分） |
| 说明 | 统一开票金额字段，不受含税类型显隐影响，始终可见 |

**5. 修改字段：含税单价**

| 项目 | 当前 | 改为 |
|------|------|------|
| 小数位数 | 2 位 | **4 位** |

**6. 含税小计 — 条件显隐**

| 项目 | 配置 |
|------|------|
| 显隐规则 | 可见当 `主表.fd_col_taxType == 0`（含税） |

**7. 设备列 — 数据源过滤**

| 主表含税类型 | 可选的设备 |
|-------------|-----------|
| 含税（0） | 仅含税合同下的设备 |
| 不含税（1） | 仅不含税合同下的设备 |

过滤在明细表设备列的数据源中配置。

---

## 6. 需求对照

| 需求项 | 描述 | 对应修改项 |
|--------|------|-----------|
| a | 添加不含税单价、不含税小计，根据公式计算本次开票金额 | 第 2、3、4 项 |
| b | 修改含税单价位数为 4 位 | 第 5 项 |
| c | 选择设备带出含税类型到表单，含税展示含税价格、隐藏不含税价格；不含税反之 | 第 1、6 项 |
| d | 如同时选择含税及不含税设备，提示信息并清空后选设备 | 第 7 项（设备列过滤从源头杜绝） |

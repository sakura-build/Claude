## 1. 前置准备

### 1.1 注册小程序账号

1. 打开 [微信公众平台](https://mp.weixin.qq.com/) → 点击右上角「立即注册」
2. 选择注册类型：**小程序**
3. 填写邮箱（不能是已注册过公众号/小程序的邮箱）、设置密码
4. 登录邮箱，点击激活链接
5. 选择主体类型：

| 主体类型 | 认证费 | 可用功能 |
|---------|--------|---------|
| **个人** | 免费 | 基本功能（本程序只需要基本功能，个人即可） |
| 企业 | 300元/年 | 微信支付、获取手机号等 |

> 💡 **本程序使用「个人」主体即可**，所有功能（云数据库、云存储、云函数）均对个人开放。

6. 完成注册后，登录 [小程序管理后台](https://mp.weixin.qq.com/)
7. 在「开发管理 → 开发设置」中找到并记录 **AppID**（格式：`wxXXXXXXXXXX`）

### 1.2 完善小程序信息（审核前必须完成）

在管理后台「设置」中填写：
- **小程序名称**：如「小厨家庭菜谱」（名称需唯一，建议准备 2-3 个备选）
- **小程序头像**：建议 144×144 像素
- **小程序简介**：描述功能（如「家庭菜谱管理、点餐和采购清单工具」）
- **类目选择**：建议选择「工具 → 信息查询」（个人主体可选类目有限，此类目容易通过审核）

## 2. 环境搭建

### 2.1 下载微信开发者工具

前往 [微信开发者工具下载页](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)，选择对应系统版本下载安装。

### 2.2 打开项目

1. 启动微信开发者工具
2. 扫码登录（与注册小程序的微信账号一致）
3. 点击「导入项目」或「+」→ 选择「导入」
4. 填写：
   - **项目目录**：选择本项目的 `recipe-miniapp` 文件夹
   - **AppID**：填入在 1.1 步骤记录的 AppID（或选择「测试号」先体验）
   - **项目名称**：`家庭菜谱`
5. 点击「导入」

### 4.1 开通云开发

1. 在微信开发者工具中，点击顶部工具栏的「云开发」图标（☁️）
2. 点击「开通」按钮

4. 创建成功后，记下 **环境 ID**（格式如 `recipe-prod-xxxxx`）

## 5. 数据库初始化

### 5.1 创建数据库集合

在开发者工具中：
1. 点击「云开发」→ 切换到「数据库」标签
2. 点击「添加集合」

创建以下 2 个集合：

#### 集合 1：`recipes`（菜谱）

**权限设置**：所有用户可读，所有用户可写
> 路径：集合名右侧「...」→ 权限设置 → 选择「所有用户可读，所有用户可写」

**索引建议**（在「索引管理」中添加，非必须但建议添加以提升性能）：
- 字段：`status`，排序：升序
- 字段：`category`，排序：升序

#### 集合 2：`orders`（订单）

**权限设置**：所有用户可读，所有用户可写

**索引建议**：
- 字段：`requestedDate`，排序：升序
- 字段：`status`，排序：升序

### 5.2 设置数据库权限的重要性

> ⚠️ **必须将权限设为「所有用户可读写」**，否则无法在小程序中读写数据。
> 
> 设置方法：
> 1. 点击集合名称
> 2. 点击右上角「权限设置」
> 3. 选择「所有用户可读，所有用户可写」

## 6. 云函数部署

### 6.1 部署步骤

本项目有 3 个云函数需要部署：

```
cloudfunctions/
├── createOrder/          # 创建点餐订单
├── updateOrderStatus/    # 更新订单状态
└── getShoppingList/      # 生成采购清单
```

**部署方法**（对每个云函数重复）：

1. 在开发者工具左侧文件树中，展开 `cloudfunctions` 文件夹
2. 右键点击云函数文件夹（如 `createOrder`）
3. 选择「**上传并部署：云端安装依赖**」
4. 等待上传完成（控制台会显示进度）

> ⚠️ **必须选择「云端安装依赖」**，这样云函数所需的 `wx-server-sdk` 会自动在云端安装。

### 6.2 验证部署

1. 在「云开发 → 云函数」标签中，确认 3 个云函数都显示为「已部署」
2. 可以点击云函数名，在详情页点击「测试」进行简单测试

## 7. 本地调试

完成以上配置后，即可在开发者工具中调试运行：

### 7.1 基础测试流程

1. 点击工具栏「编译」按钮
2. 确认 4 个 Tab 页能正常切换
3. 按以下流程走通全链路：

```
① 点击「菜谱」Tab → 点右下角 + 号
② 填写菜名、选择分类、添加食材和步骤 → 保存
③ 在菜谱列表中点击刚创建的菜谱 → 查看详情
④ 点击「点这道菜」→ 选择餐别（早/中/晚）→ 确认点餐
⑤ 切换到「点餐」Tab → 查看订单，操作状态流转
⑥ 从「首页」点击「采购清单」→ 选择日期查看聚合食材
```

### 7.2 TabBar 图标

默认 TabBar 使用纯文字。如需添加图标：
1. 准备 8 张 PNG 图标（每个 Tab 有普通态和选中态，共 4 个 Tab × 2 = 8 张）
2. 图标尺寸建议 40×40 像素（实际显示 81×81 会缩放）
3. 放入 `miniprogram/images/` 目录
4. 在 `miniprogram/app.json` 的 `tabBar.list` 中为每个 Tab 添加 `iconPath` 和 `selectedIconPath`

### 7.3 真机预览

1. 点击开发者工具顶部的「预览」按钮
2. 扫码在手机上体验真实效果
3. 可以在手机上测试拍照上传、日期选择等原生功能

## 8. 提交审核与上线

> ⚠️ 审核前确保已注册小程序账号、完善小程序信息、设置好类目。

### 8.1 上传代码

1. 在开发者工具中点击右上角「上传」按钮
2. 填写版本号（如 `1.0.0`）和版本描述（如「首次发布：菜谱管理、点餐、采购清单」）
3. 点击「上传」，代码会被上传到微信服务器

### 8.2 提交审核

1. 登录 [小程序管理后台](https://mp.weixin.qq.com/)
2. 左侧菜单 → 「管理」→ 「版本管理」
3. 在「开发版本」中找到刚上传的版本 → 点击「提交审核」
4. 填写审核信息：

| 字段 | 填写内容 |
|------|----------|
| 功能页面 | 首页、菜谱列表、菜谱详情、菜谱编辑、点餐页、订单列表、采购清单、个人中心 |
| 所在类目 | 工具 → 信息查询 |
| 测试账号 | 无需（本小程序无登录功能，使用云开发 openid 自动识别） |
| 功能截图 | 上传 3-5 张主要页面截图 |

5. 提交后等待审核（通常 **1-7 个工作日**）

### 8.3 常见审核被拒原因及应对

| 被拒原因 | 解决方法 |
|---------|---------|
| 类目不匹配 | 选择「工具 → 信息查询」类目 |
| 功能不完整 | 确保所有页面数据能正常加载（页面不要空白） |
| 服务异常 | 确保云函数已部署、数据库权限已设置 |
| 需测试账号 | 审核备注中说明「无需登录，使用云开发自动鉴权」 |

### 8.4 发布上线

1. 审核通过后会收到微信通知
2. 登录管理后台 → 版本管理 → 「审核版本」
3. 点击「发布」
4. 发布后用户即可在微信中搜索到你的小程序

> 💡 发布后可选择「分阶段发布」（灰度发布），先给部分用户使用，确认无问题后再全量发布。

### 8.5 后续更新

每次更新代码后：
1. 开发者工具 → 上传（新版本号如 `1.0.1`）
2. 管理后台 → 提交审核 → 审核通过 → 发布

## 9. 常见问题

### Q1：编译报错「cloud.init() not called」
> 确认 `app.js` 中 `env` 已改为你的云环境 ID，且云开发已开通。

### Q2：数据加载不出来，页面空白
> 检查数据库集合权限是否设为「所有用户可读写」。如果没有创建 `recipes` 和 `orders` 集合，需要手动创建。

### Q3：云函数调用报错
> 确认云函数已「上传并部署」。在「云开发 → 云函数」中查看云函数状态是否为「已部署」。

### Q4：图片上传失败
> 确认 `app.js` 中 `wx.cloud.init()` 已正确初始化。检查云存储是否已启用（云开发控制台 → 存储）。

### Q5：手机上样式错乱
> `rpx` 是响应式单位，在不同屏幕宽度自动适配。如果遇到问题，确认 `app.wxss` 中的 CSS 变量定义未被覆盖。

### Q6：如何让家人都能使用？
> 小程序上线后，将小程序分享到家庭微信群即可。每个用户由云开发根据 `openid` 自动识别，数据共享全家可见。

### Q7：TabBar 图标从哪里获取？
> 可以从 [iconfont](https://www.iconfont.cn/) 下载 4 组 PNG 图标（每组 2 张：普通态 + 选中态），或使用开发者工具的可视化 TabBar 编辑器设置。

### Q8：如何区分谁点的菜？
> 点餐时填写「点餐人」昵称（如「妈妈」「小明」）。昵称会保存在手机本地，下次自动填充。
